Query-Based Book Recommendation System

Introduction

Welcome! I want to share the story behind my Query-Based Book Recommendation System. Personally speaking, I am an avid reader, but I constantly found myself struggling to find a book based on a specific "vibe" or abstract concept. I wanted to build a semantic search platform that allows people to describe what they want to read in plain English, and instantly get the perfect recommendation.

The Problem I Wanted to Solve

Traditional keyword-based search is fundamentally broken for creative discovery. If you search a library database for "a story about isolation in deep space," standard search engines look for those exact keywords. But what if the best matching book's blurb says "a lone astronaut adrift in the cosmos"? The meaning is identical, but the words are entirely different. I needed a system that understands semantics (meaning) rather than just syntax (spelling).

How It Works (Under the Hood)

To bridge this semantic gap, I leveraged Sentence Transformers, specifically the all-MiniLM-L6-v2 model. Here is the step-by-step flow:

  • Vectorization: I pre-computed high-dimensional vector embeddings for the summaries and metadata of over 7,000 books. In this mathematical space, books with similar themes are located closer together.
  • Query Processing: When a user types a natural-language query, the system converts their sentence into the exact same vector space.
  • Retrieval: Using Cosine Similarity, the system calculates the angle between the user's query vector and all the book vectors. The closest vectors represent the best semantic matches.

I wrapped all of this logic into a sleek, interactive frontend using Streamlit and deployed it on the Streamlit Community Cloud so anyone could access it from their browser.

Challenges & Learnings

Working with vector embeddings at scale introduces interesting performance challenges. Calculating cosine similarity across 7,000+ high-dimensional vectors on the fly can become a bottleneck. I spent a significant amount of time optimizing the tensor operations in PyTorch and caching the embeddings to ensure the search felt instantaneous to the user. It taught me a lot about memory management and computational efficiency in Python.

Final Thoughts

This project successfully proved to me how accessible and powerful modern NLP tools have become. Transforming a vague feeling into a concrete book recommendation feels a bit like magic, and it's a testament to the capabilities of retrieval-augmented generation (RAG) concepts when applied to search.
