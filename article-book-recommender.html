Project 06 / Semantic retrieval

Query-Based Book Recommendation System

2025 / Meaning-aware discovery

A semantic search application that converts natural-language reading intent into ranked books using precomputed description embeddings and cosine similarity.

~5,200 booksClean descriptions
384 dimensionsMiniLM embeddings
1–20 resultsConfigurable Top-K

Problem and search behaviour

Readers often remember a theme rather than a title: “a mystery in a small town,” “philosophy for someone new to the subject,” or “isolation in deep space.” Keyword search is brittle in this setting because the words in the query may never appear in a book’s description even when the meaning matches.

The project treats discovery as semantic retrieval. The query and every book description are encoded with the same sentence model, allowing the system to compare meaning in a shared vector space rather than count overlapping terms.

Dataset preparation

The source dataset contains roughly 7,000 records with titles, authors, descriptions, categories, cover URLs, publication years, ratings, and other metadata. Records without usable descriptions are removed, leaving approximately 5,200 entries for retrieval.

Embedding generation is deliberately separated from the interactive application. A preparation script loads all-MiniLM-L6-v2, encodes every description with a progress bar, and saves the resulting matrix as book_embeddings.npy. The cleaned dataframe is saved alongside it. This turns a repeated model workload into a one-time indexing step.

Retrieval pipeline

  • Load: Streamlit caches the sentence model as a resource and the dataframe/embedding matrix as data.
  • Encode: only the incoming user query is encoded during interaction.
  • Compare: scikit-learn computes cosine similarity between the query vector and the complete embedding matrix.
  • Filter: an optional category filter narrows the candidate dataframe after similarity scores are attached.
  • Rank: candidates are sorted descending by similarity and truncated to the requested Top-K.
  • Present: results show cover art, title, author, rating, year, category, score, and an expandable description.

Why precompute embeddings

Encoding every book description on every interaction would make response time proportional to both corpus size and transformer inference cost. Precomputation changes the query path to one transformer call plus matrix similarity. At this corpus size, brute-force cosine similarity is simple, transparent, and fast enough while preserving exact ranking.

The trade-off is index freshness. When descriptions change or books are added, the embedding file must be regenerated. A larger or frequently changing catalogue would benefit from a vector database, incremental indexing, and versioned embeddings.

Interface decisions

The sidebar exposes Top-K and a curated category filter. Results are presented as metadata-rich rows rather than only a list of titles. Displaying the similarity score is useful for debugging and demonstration, although a consumer-facing product might hide it or translate it into a less technical relevance explanation.

Cover images are loaded from dataset URLs, while missing covers fall back to a placeholder. Descriptions are collapsed by default so the user can scan multiple recommendations without facing a wall of text.

Evaluation and limitations

Cosine similarity provides a ranking but does not establish that the recommendations are useful. A robust evaluation set would pair realistic queries with human relevance labels and report Precision@K, Recall@K, NDCG, and qualitative failure categories.

The model only sees descriptions, so it may miss signals contained in reviews, series information, audience, writing style, or reader behaviour. Category filtering is implemented as substring matching, which is convenient but not a formal taxonomy. The project is semantic search rather than collaborative recommendation, and the article uses that distinction deliberately.

Most important lesson

For small corpora, the simplest retrieval architecture can be the best one. Precomputed embeddings plus exact cosine similarity made the system easy to inspect, test, and explain before introducing approximate indexes.

Next steps

A stronger version would add a labelled query set, hybrid lexical/vector retrieval, diversity-aware reranking, query history, embedding-version checks, and filters driven by normalized metadata. Moving the matrix into Qdrant or another ANN service would become valuable once corpus size or update frequency justifies the added operational complexity.
