Case file 01 / Multimodal retrieval

Lumina Distributed Multimodal Semantic Search Engine

Feb 2026 — Mar 2026

Introduction

Hey there! I want to talk to you about Lumina, my distributed multimodal semantic search engine. Personally speaking, the boundary between text and images has always fascinated me. We process the world visually, but we communicate primarily through language. I built Lumina to bridge that gap—allowing users to search through massive datasets of images using natural, descriptive language.

The Problem I Wanted to Solve

Imagine you have a library of 8,000+ photos. If you want to find "a dog running on the beach at sunset," traditional image search requires that someone meticulously tagged that photo with those exact words beforehand. This manual tagging is unscalable and highly subjective. I wanted a system that could "look" at the images, understand their content, and instantly match them to text queries without relying on pre-existing metadata.

How It Works (Under the Hood)

Lumina relies on some incredible deep learning architecture:

  • CLIP (Contrastive Language-Image Pre-Training): I utilized OpenAI's CLIP-VIT-B-32 model. CLIP is uniquely trained to project both text and images into the exact same mathematical vector space. This means the vector for the text "a cat" is located right next to the vector representing a picture of a cat.
  • Qdrant Vector Database: To handle the 8,000+ image vectors, I deployed Qdrant. It's incredibly fast at performing nearest-neighbor searches in high-dimensional spaces.
  • FastAPI & MPS Acceleration: I built the ingestion backend using FastAPI. To speed up the heavy lifting of generating embeddings, I engineered the system to utilize Apple's MPS (Metal Performance Shaders) for hardware acceleration.

When you type a query, the FastAPI backend uses CLIP to turn your text into a vector, asks Qdrant for the closest image vectors, and serves the results back to a streamlined Streamlit dashboard.

Challenges & Learnings

Scaling up to 8,000+ images isn't just a matter of running a loop. My initial ingestion scripts kept timing out or crashing due to memory limits. I had to learn how to implement robust batch upserts, carefully chunking the data and monitoring the API health during the massive upload process. Getting the MPS acceleration properly configured for PyTorch on Mac architecture was also a steep, but very rewarding, learning curve.

Final Thoughts

Lumina is a project I am incredibly proud of. It shifted my perspective from building standalone ML scripts to architecting scalable, distributed systems. Being able to type a complex sentence and instantly watch the system pull up the exact image I was imagining still feels like science fiction.
